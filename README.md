


# 📖 Описание проекта: Roguelike "Подземелье Координат" (v2.0)

## 🚀 Введение

**"Подземелье Координат"** — это пошаговая roguelike-игра, написанная на чистом JavaScript с использованием библиотеки ROT.js для процедурной генерации и работы с полем зрения (FOV). Проект представляет собой гибрид классического dungeon-crawler'а и текстового приключения, где игрок исследует бесконечный мир, спускается в процедурно генерируемые подземелья, торгует в городах и выполняет сюжетные квесты.

### Ключевые особенности архитектуры v2.0:
1.  **Гибридная графика:** Отказ от чистого ASCII в пользу спрайтовых тайлсетов (PNG) с программной окраской (`source-atop`), что позволяет сохранять ретро-стиль, но добавляет визуальную глубину и поддержку больших существ (боссы 2x2).
2.  **Детерминированный мир:** Вся генерация (ландшафт, имена, типы подземелий, расположение лестниц) привязана к координатам через seeded RNG. Это гарантирует, что мир остается неизменным при повторных посещениях, но при этом бесконечен.
3.  **Сложная система квестов:**
    *   *Процедурные:* Бесконечные задания на охоту, сбор и разведку.
    *   *Сюжетные цепочки:* Детерминированная линия, связывающая города мира в единую историю.
    *   *Текстовые (Twine):* Интеграция внешних HTML-файлов с интерактивными историями через Iframe и `postMessage`.
4.  **Механика выносливости:** Глобальный ресурс, ограничивающий перемещение по карте мира и требующий планирования маршрутов и отдыха в городах.
5.  **Модульность:** Код разделен на независимые IIFE-модули (Core, World, Logic, UI), что упрощает масштабирование и отладку.

## 🔄 Поток данных (Data Flow)

1. **Инициализация**: 
   `GameModule.init()` → Загрузка тайлсетов (`TilesetRenderer`) → Генерация стартовой позиции на глобальной карте.

2. **Глобальный режим**:
   Игрок двигается по чанкам → `GlobalMapModule` проверяет POI → Вход в Город или Подземелье.

3. **Загрузка локации**:
   `loadCityLevel()` / `loadDungeonLevel()` → Генерация карты (`MapModule` + `DungeonGenerator`) → Спавн сущностей (`EntityModule`) → Расчет лестниц.

4. **Игровой цикл (Ход)**:
   `handleInput()` → Движение игрока → Проверка столкновений (Враги/NPC/Предметы/Лестницы) → 
   Ход NPC (`moveNpcs`) → Ход врагов (`moveEnemies`, AI A*) → 
   Обработка эффектов (`EffectSystem`) → Проверка смерти → Отрисовка (`RenderModule`).

5. **Взаимодействие**:
   - **Бой**: `CombatModule` (ближний/дальний) → Обновление HP → Loot.
   - **Торговля**: Открытие окна магазина → Покупка/Продажа → Обновление инвентаря.
   - **Квесты**: 
     - Стандартные: `QuestSystemModule` проверяет условия (убийство, подбор).
     - Сюжетные: `QuestChainModule` выдает задания в конкретных городах.
     - Текстовые: NPC запускает Iframe с Twine-историей → Возврат награды через `postMessage`.

## 🔗 Взаимодействие модулей

- **game.js** является центральным хабом, вызывая методы всех остальных модулей.
- **render.js** зависит от **tileset_renderer.js** для графики и **ROT.js** для FOV.
- **entity.js** использует **data.js** для шаблонов и **worldCurve.js** для масштабирования силы.
- **quest_chain.js** взаимодействует с **globalMap.js** для поиска городов и **game.js** для управления состоянием квестов.
- **combat.js** интегрирован с **effect_system.js** для применения баффов от зелий и оружия.

# 📖 I. Руководство моддера: Как добавлять новые предметы в игру

Добро пожаловать! Этот гайд поможет тебе добавить в игру новое оружие, броню, зелья, еду или уникальные артефакты. Тебе **не нужно** быть программистом, достаточно уметь копировать, вставлять и следить за запятыми.

## 🛠️ Что тебе понадобится?
1. **Текстовый редактор:** Блокнот не подойдет (он может испортить код). Скачай **Notepad++**, **VS Code** или **Sublime Text**.
2. **Файл `data.js`**: Вся база предметов находится именно здесь. Открой его в редакторе.

## 🟢 ЧАСТЬ 1. Добавление обычного предмета (Оружие, Броня, Зелья, Еда)

Все обычные предметы лежат в массиве `ITEM_TYPES`. 
Найди в `data.js` строку `const ITEM_TYPES = [` — ниже идет список всех предметов.

### Шаг 1. Выбери "донора"
Чтобы не писать с нуля, найди предмет, похожий на тот, что ты хочешь создать, и скопируй его блок целиком (от `{` до `}`).

**Пример: Допустим, мы хотим добавить "Ядовитый гриб" (Еда/Лечение).**
Найдем обычный "Хлеб и сыр" и скопируем его.

### Шаг 2. Измени параметры под себя
Вот шпаргалка по параметрам, которые можно менять:

```javascript
{ 
    type: "food",             // ТИП: weapon (оружие), armor (броня), food (еда), potion_hp (лечение), potion_str (сила), potion_def (защита), book (книга), scroll_teleport (свиток).
    char: "%",                // СИМВОЛ: Картинка на карте. "%" - это еда. (Список символов см. в Части 3).
    color: "#8BC34A",         // ЦВЕТ: HEX-код цвета. #8BC34A - зеленый. Можно менять на любые.
    baseName: "Ядовитый гриб",// ИМЯ: Базовое название. Игра сама может добавлять прилагательные (например, "Свежий Ядовитый гриб").
    effect: "heal",           // ЭФФЕКТ: "heal" (лечит), "buff_atk" (дает силу), "buff_def" (дает защиту).
    val: [5, 15],             // ЗНАЧЕНИЕ: От и До. [5, 15] значит, что гриб вылечит от 5 до 15 HP.
    gender: "he",             // РОД: "he" (он - гриб), "she" (она - поганка), "it" (оно - зелье). Нужно для красивых имен!
    plural: false             // МНОЖЕСТВЕННОЕ ЧИСЛО: true если предмет во множественном числе (например, "Наголенники").
}
```

### Шаг 3. Вставь новый предмет в список
Вставь свой измененный блок **перед** закрывающей скобкой `];` у массива `ITEM_TYPES`.

⚠️ **ВАЖНОЕ ПРАВИЛО:** После каждой закрывающей фигурной скобки `}` должна стоять **запятая** `,`. Перед самой последней `];` запятая не нужна.

**Как это должно выглядеть в коде:**
```javascript
        // ... старые предметы ...
        { 
            type: "food", 
            char: "%", 
            color: "#8BC34A", 
            baseName: "Хлеб и сыр", 
            effect: "heal", 
            val: [5, 10],
            gender: "he", 
            plural: false
        }, // <--- ЗАПЯТАЯ ОБЯЗАТЕЛЬНА!

        // === ТВОЙ НОВЫЙ ПРЕДМЕТ ===
        { 
            type: "food", 
            char: "%", 
            color: "#9C27B0", // Фиолетовый цвет для ядовитости
            baseName: "Спорынья", 
            effect: "heal", 
            val: [2, 8], // Лечит мало, но это же яд!
            gender: "she", // Она (спорынья)
            plural: false
        } // <--- БЕЗ ЗАПЯТОЙ, так как это последний элемент!
    ];
```

## 🟣 ЧАСТЬ 2. Добавление Уникального предмета (Артефакты)

Уникальные предметы (Уники) не генерируются случайно из обычных. Они имеют жесткие статы, уникальное имя и префикс. Они часто используются в квестах.

Ищи в `data.js` массив `UNIQUE_ITEM_TEMPLATES = [`.

### Отличия Уников от обычных предметов:
1. У них есть **`id`** (уникальный идентификатор, нужен для квестов).
2. Вместо `val` у них используется **`atk`** (для оружия) или **`def`** (для брони).
3. Есть **`uniquePrefix`** и **`desc`** (описание).

### Пример: Создаем "Кольцо Вампира"
```javascript
        {
            id: "unique_ring_vampire",      // ID: Придумай уникальный латинский текст без пробелов.
            baseType: "armor",              // БАЗОВЫЙ ТИП: weapon или armor. (Кольца и амулеты в игре считаются броней).
            baseName: "Кольцо Крови",       // ИМЯ: Как называется предмет.
            uniquePrefix: "Проклятое ",     // ПРЕФИКС: Будет написано перед именем. (Обрати внимание на пробел в конце!)
            def: [3, 5],                    // СТАТЫ: [Мин, Макс] защиты. 
            // Если это оружие, пиши atk: [15, 20], вместо def.
            color: "#e74c3c",               // ЦВЕТ: Красный (кровавый).
            desc: "Пьет здоровье врагов."   // ОПИСАНИЕ: Для лора и инспектора.
        },
```
*Не забудь поставить запятую после `}`, если это не последний предмет в списке!*

## 🎨 ЧАСТЬ 3. Шпаргалка по Символам (`char`) и Цветам

Символ (`char`) — это то, какую картинку из тайлсета (PNG) игра подставит на карту. 
**Ты не можешь написать `char: "меч"`.** Ты должен использовать один из символов из реестра.

**Популярные символы для предметов:**
*   **Оружие:** `/` (Меч), `P` (Топор), `)` (Булава), `*` (Кинжал), `Y` (Копье), `(` (Лук), `=` (Арбалет), `|` (Посох).
*   **Броня:** `]` (Кожа), `[` (Кольчуга), `}` (Щит), `{` (Плащ), `H` (Шлем), `v` (Перчатки).
*   **Расходники:** `!` (Зелье), `+` (Эликсир), `%` (Еда/Хлеб), `~` (Мясо), `?` (Книга), `$` (Золото).

**Популярные HEX-цвета (вставляй в `color: "..."`):**
*   🔴 Красный: `#f44336` или `#e74c3c`
*   🟠 Оранжевый: `#ff9800`
*   🟡 Золотой: `#FFD700`
*   🟢 Зеленый: `#4CAF50`
*   🔵 Синий: `#2196F3`
*   🟣 Фиолетовый: `#9C27B0` или `#9b59b6`
*   ⚪ Серый/Серебро: `#9E9E9E` или `#B0BEC5`

## 📝 ЧАСТЬ 4. Секрет механики имен (Род и Прилагательные)

Игра сама генерирует крутые имена, вроде *"Мифриловый Меч"* или *"Ржавая Кольчуга"*. Чтобы это работало, ты должен правильно указать **`gender`** (род) в обычном предмете.

*   `"he"` (Он) — Меч, Топор, Кинжал, Эликсир, Хлеб, Гриб.
    *   *Пример:* Меч -> *Острый* Меч.
*   `"she"` (Она) — Булава, Кольчуга, Броня, Поганка, Кость.
    *   *Пример:* Кольчуга -> *Ржавая* Кольчуга.
*   `"it"` (Оно) — Зелье, Копье, Ядро.
    *   *Пример:* Зелье -> *Мутное* Зелье.
*   `"plural"` (Множ. число) — Наголенники, Перчатки.
    *   *Пример:* Перчатки -> *Кожаные* Перчатки (тут `plural: true`).

## ⚠️ Чек-лист перед сохранением (Чтобы игра не сломалась)

1. **Все скобки закрыты?** У каждого `{` есть своя `}`.
2. **Запятые на месте?** Между предметами в списке обязательно должны быть запятые.
   * ❌ Ошибка: `} {` (Нет запятой)
   * ✅ Правильно: `}, {`
3. **Кавычки прямые?** Используй только обычные кавычки `" "` и `' '`. Word и некоторые мессенджеры заменяют их на «елочки» `" "`, из-за этого код выдаст ошибку!
4. **Нет лишней запятой в конце?** Перед закрывающим `];` запятая не нужна.

### 💡 Заметки для тебя (как для разработчика):
1. Этот гайд намеренно **не затрагивает** добавление *новых спрайтов* (рисование в PNG, правка `sprite_registry.js` и `tileset_renderer.js`). Для неспециалиста это пока слишком сложно. Пусть они используют существующие иконки, меняя им цвет (`color`) — система `source-atop` в рендерере это отлично поддерживает.
2. Cледующим шагом напишем гайд по **созданию новых видов оружия и брони.**

Отличный выбор! Оружие и броня — это основа любого RPG-лута. В нашей игре система генерации имен сама сделает твой новый предмет красивым (например, *"Ржавый Боевой Молот"* или *"Мифриловая Аркебуза"*), тебе нужно только задать базу.

Этот гайд научит тебя добавлять **обычные** виды экипировки, которые будут случайно выпадать из врагов и продаваться в магазинах.

# ⚔️🛡️ II. Руководство моддера: Создание нового Оружия и Брони

Вся база предметов лежит в файле **`data.js`**. 
Тебе нужен массив `const ITEM_TYPES = [ ... ];`.

## 🟢 ШАГ 1. Выбираем иконку (Спрайт)
Игра использует готовые картинки из тайлсетов. Чтобы не рисовать новые, мы возьмем существующую иконку, но **поменяем ей цвет** и дадим уникальное имя.

Вот список ID иконок, которые ты можешь использовать в коде (через функцию `getChar('ID')`):

**Оружие:**
*   `ITEM_SWORD` (Меч)
*   `ITEM_AXE` (Топор)
*   `ITEM_MACE` (Булава)
*   `ITEM_DAGGER` (Кинжал)
*   `ITEM_SPEAR` (Копьё)
*   `ITEM_BOW` (Лук)
*   `ITEM_CROSSBOW` (Арбалет)
*   `ITEM_STAFF` (Посох)

**Броня:**
*   `ITEM_ARMOR_LEATHER` (Кожаная броня)
*   `ITEM_ARMOR_CHAIN` (Кольчуга)
*   `ITEM_SHIELD` (Щит)
*   `ITEM_GREAVES` (Наголенники)
*   `ITEM_CLOAK` (Плащ)
*   `ITEM_HELMET` (Шлем)
*   `ITEM_GLOVES` (Перчатки)

---

## 🟢 ШАГ 2. Добавляем предмет в `data.js`

Открой `data.js`, найди список `ITEM_TYPES` и вставь свой блок **перед** закрывающей скобкой `];`. Не забудь поставить запятую после предыдущего предмета!

### 🗡️ Вариант А: Оружие Ближнего Боя
```javascript
        { 
            type: "weapon",                 // Тип: оружие
            char: getChar('ITEM_MACE'),     // Иконка: булава (можно любую из списка)
            color: "#795548",               // Цвет: коричневый (дерево/железо)
            baseName: "Боевой молот",       // Базовое имя
            stat: "atk",                    // Качает атаку
            val: [4, 9],                    // Урон: от 4 до 9 (до генерации редкости)
            gender: "he",                   // Род: ОН (молот)
            plural: false,                  // Единственное число
            meleeType: true,                // ВАЖНО: Это ближний бой!
            range: 1                        // Дальность: 1 клетка
        },
```

### 🏹 Вариант Б: Дистанционное Оружие (Стрелковое)
Для луков и арбалетов добавляются параметры `maxAmmo` (максимум патронов/стрел).
```javascript
        { 
            type: "weapon", 
            char: getChar('ITEM_CROSSBOW'), 
            color: "#607D8B",               // Цвет: темное железо
            baseName: "Тяжелый арбалет", 
            stat: "atk", 
            val: [6, 12],                   // Стрелковое бьет сильнее, но имеет патроны
            gender: "he", 
            plural: false, 
            meleeType: false,               // ВАЖНО: Это ДАЛЬНИЙ бой!
            range: 12,                      // Дальность стрельбы (клеток)
            maxAmmo: 15                     // Максимум стрел/болтов перед перезарядкой
        },
```

### 🛡️ Вариант В: Броня (Защита)
У брони нет `meleeType` и `range`. Вместо `atk` используется `def`.
```javascript
        { 
            type: "armor",                  // Тип: броня
            char: getChar('ITEM_SHIELD'),   // Иконка: щит
            color: "#FFC107",               // Цвет: золото/латунь
            baseName: "Тарgetный щит",      // Базовое имя
            stat: "def",                    // Качает защиту
            val: [3, 6],                    // Защита: от 3 до 6
            gender: "he",                   // Род: ОН (щит)
            plural: false                   
        },
```

---

## 🟢 ШАГ 3. Грамматика и Род (Чтобы имена звучали красиво)

Игра сама генерирует прилагательные (Ржавый, Мифриловый, Острый). Чтобы не было ошибок вроде *"Ржавая Меч"*, нужно правильно указать **`gender`** (род слова):

*   **`"he"` (Он)** — Меч, Топор, Молот, Кинжал, Лук, Арбалет, Щит, Плащ, Шлем, Посох.
    *   *Пример:* `gender: "he"` ➔ *"Острый Боевой молот"*
*   **`"she"` (Она)** — Булава, Кольчуга, Броня, Секира, Аркебуза.
    *   *Пример:* `gender: "she"` ➔ *"Тяжелая Секира"*
*   **`"it"` (Оно)** — Копьё, Зелье, Ядро.
    *   *Пример:* `gender: "it"` ➔ *"Каленое Копьё"*
*   **`"plural"` (Они / Множественное число)** — Наголенники, Перчатки, Латы.
    *   *Пример:* `gender: "plural", plural: true` ➔ *"Кожаные Перчатки"*

---

## 🟢 ШАГ 4. Как сбалансировать предмет? (Цифры `val`)

Игра сама масштабирует силу предмета в зависимости от глубины подземелья. Но **базовый разброс** `val: [мин, макс]` задает "вес" предмета в мире.

**Ориентир для Оружия (`atk`):**
*   `[1, 3]` — Слабое (Кинжал, посох ученика)
*   `[3, 6]` — Среднее (Меч, лук)
*   `[5, 9]` — Сильное (Топор палача, арбалет)
*   `[8, 14]` — Эпическое (Тяжелое двуручное оружие)

**Ориентир для Брони (`def`):**
*   `[1, 2]` — Слабая (Тряпки, кожаный нагрудник)
*   `[2, 4]` — Средняя (Кольчуга, щит)
*   `[4, 7]` — Сильная (Латы, башенный щит)
*   `[6, 10]` — Эпическая (Адамантиновые латы)

💡 **Лайфхак:** Если ты хочешь сделать медленное, но мощное оружие, дай ему большой `val` (например, `[8, 15]`), но оставь `range: 1`. Если хочешь сделать снайперское оружие — делай `range: 20`, но `val` поменьше.

---

## ⚠️ Чек-лист перед сохранением

1. **Запятые:** После каждой `}` должна стоять `,`. Перед самой последней `];` запятая не нужна.
2. **Скобки:** У каждого `{` есть закрывающая `}`. У массива `[` есть `]`.
3. **Кавычки:** Используй только прямые кавычки `" "` и `' '`.
4. **Дистанционка:** Если делаешь лук/арбалет/посох, **обязательно** пиши `meleeType: false` и `maxAmmo: число`. Иначе игра будет считать это дубиной для ближнего боя!

# III. 👹 Руководство моддера: Создание новых монстров

Монстры — это сердце подземелий. В этом гайде ты научишься добавлять новых врагов: от слабых крыс до могучих демонов. Ты сможешь задать им уникальные статы, иконку, добычу и даже "прописку" на определённой глубине.

---

## 🟢 ШАГ 1. Выбираем иконку (Спрайт)

Все иконки монстров хранятся в файле **`sprite_registry.js`** в разделе `// 4. ВРАГИ (ENEMY_TYPES)`.

Вот список ID, которые ты можешь использовать:

**Свободные для использования (уже есть в игре):**
*   `ENEMY_RAT` (Крыса) — `r`
*   `ENEMY_GOBLIN` (Гоблин) — `g`
*   `ENEMY_WOLF` (Волк) — `w`
*   `ENEMY_BANDIT` (Бандит) — `b`
*   `ENEMY_SKELETON` (Скелет) — `s`
*   `ENEMY_SLIME` (Слизень) — `j`
*   `ENEMY_ORC` (Орк) — `k`
*   `ENEMY_ZOMBIE` (Зомби) — `z`
*   `ENEMY_HARPY` (Гарпия) — `h`
*   `ENEMY_GHOST` (Призрак) — `G`
*   `ENEMY_VAMPIRE` (Вампир) — `V`
*   `ENEMY_TROLL` (Тролль) — `t`
*   `ENEMY_LICH` (Лич) — `L`
*   `ENEMY_GOLEM` (Голем) — `M`
*   `ENEMY_DRAGON` (Дракон) — `q`

💡 **Лайфхак:** Ты можешь взять *любую* иконку из реестра и перекрасить её через параметр `color`. Так, один и тот же спрайт "Гоблина" может стать зелёным орком, красным демоном или синим ледяным элементалем — просто меняй HEX-цвет!

---

## 🟢 ШАГ 2. Добавляем монстра в `data.js`

Открой файл **`data.js`** и найди массив `const ENEMY_TYPES = [ ... ];`. Вставь свой блок **перед** закрывающей скобкой `];`.

### 📋 Шаблон нового монстра:
```javascript
        { 
            name: "Ледяной элементаль",      // ИМЯ: Как называется враг
            char: getChar('ENEMY_SLIME'),    // ИКОНКА: Берём спрайт из реестра
            color: "#4FC3F7",                // ЦВЕТ: HEX-код (голубой для льда)
            hp: [30, 45],                    // ЗДОРОВЬЕ: [мин, макс]
            atk: [5, 8],                     // АТАКА: [мин, макс]
            def: [2, 4],                     // ЗАЩИТА: [мин, макс]
            lootType: "gold",                // ДОБЫЧА: "gold", "food" или "weapon"
            speed: 8                         // СКОРОСТЬ: 1-20 (10 = скорость игрока)
        },
```

---

## 🟢 ШАГ 3. Разбор параметров (Шпаргалка)

### 📛 `name` — Имя монстра
Просто строка. Игра использует его в логах боя и в квестах на охоту (HUNT/BOUNTY).
*   *Примеры:* `"Теневой убийца"`, `"Болотная тварь"`, `"Огненный имп"`

### 🎨 `char` + `color` — Внешний вид
*   `char` — ID из реестра спрайтов (через `getChar('ID')`).
*   `color` — HEX-код цвета. Игра **перекрасит** спрайт в этот цвет.
*   *Популярные цвета:*
    *   🔴 Красный/Демон: `#C62828`, `#e74c3c`
    *   🟢 Яд/Слизь: `#4CAF50`, `#00BCD4`
    *   🔵 Лёд/Магия: `#4FC3F7`, `#2196F3`
    *   🟣 Нежить/Тьма: `#7B1FA2`, `#9C27B0`
    *   ⚪ Призрак/Кость: `#B0BEC5`, `#CFD8DC`
    *   🟤 Земля/Грязь: `#795548`, `#8D6E63`

### ❤️ `hp: [мин, макс]` — Здоровье
Среднее значение берётся за основу, потом масштабируется сложностью подземелья.

### ⚔️ `atk: [мин, макс]` — Атака
Средний урон, который враг наносит за удар (до вычета защиты игрока).

### 🛡️ `def: [мин, макс]` — Защита
Сколько урона враг поглощает. **Важно:** защита растёт очень медленно по формуле игры, чтобы игрок всегда наносил хотя бы 1 урон.

### 💰 `lootType` — Что выпадает после смерти
*   `"gold"` — Золотые монеты (чаще всего)
*   `"food"` — Еда (хлеб, мясо)
*   `"weapon"` — Оружие или броня (шанс 40%)
*   *(Если не указать — ничего не выпадет)*

### ⚡ `speed` — Скорость хода
В игре действует система энергии. Порог игрока = **10**.
*   `speed: 10` — ходит в ногу с игроком (стандарт)
*   `speed: 12-15` — очень быстрый (гарпии, волки)
*   `speed: 3-5` — медленный (слизни, зомби, голем)
*   `speed: 8-9` — чуть медленнее игрока (скелеты, орки)

---

## 🟢 ШАГ 4. Привязка к глубине (Где будет появляться?)

В файле **`entity.js`** есть функция `getAvailableEnemies(depth)`, которая решает, **каких монстров спавнить на какой глубине**.

Найди эту функцию и добавь своего монстра в нужный фильтр:

```javascript
    function getAvailableEnemies(depth) {
        if (depth <= 2) {
            return DataModule.ENEMY_TYPES.filter(e => 
                ["Крыса", "Гоблин", "Волк", "Слизень", "ТВОЙ_МОНСТР"].includes(e.name)
            );
        } else if (depth <= 6) {
            return DataModule.ENEMY_TYPES.filter(e => 
                ["Бандит", "Скелет", "Орк-разведчик", "Зомби", "Гарпия", "Призрак"].includes(e.name)
            );
        } else {
            return DataModule.ENEMY_TYPES.filter(e => 
                ["Тролль", "Вампир", "Лич", "Голем", "Демон", "Дракон", "ТВОЙ_МОНСТР"].includes(e.name)
            );
        }
    }
```

💡 **Совет:** Если хочешь, чтобы монстр появлялся **на всех глубинах** — добавь его имя во все три списка. Если только на средних — только в средний.

---

## 🟢 ШАГ 5. Балансировка (Таблица ориентиров)

Чтобы не делать монстра слишком сильным или слабым, используй эту шпаргалку:

### 🐀 Слабые враги (Глубина 1-3)
| Параметр | Значение |
|---|---|
| HP | 8-25 |
| ATK | 1-3 |
| DEF | 0-1 |
| Speed | 8-12 |
| *Примеры:* Крыса, Гоблин, Слизень |

### ⚔️ Средние враги (Глубина 4-7)
| Параметр | Значение |
|---|---|
| HP | 20-50 |
| ATK | 3-7 |
| DEF | 1-3 |
| Speed | 8-10 |
| *Примеры:* Скелет, Зомби, Гарпия, Призрак |

### 👹 Сильные враги (Глубина 8+)
| Параметр | Значение |
|---|---|
| HP | 50-150 |
| ATK | 7-15 |
| DEF | 2-8 |
| Speed | 8-10 |
| *Примеры:* Тролль, Лич, Голем, Дракон |

⚠️ **Важно:** Игра сама масштабирует статы в зависимости от глубины через `WorldCurveModule`. Твои значения — это **база**, от которой всё считается. Не завышай цифры в 2-3 раза, иначе монстр станет неубиваемым!

```javascript
            color: "#4A148C", 
            hp: [70, 95],      // Очень много HP
            atk: [10, 14],     // Больной урон
            def: [4, 6],       // Хорошая защита
            lootType: "weapon", // Всегда роняет оружие
            speed: 9 
        },
```

# IV. 📜 Руководство моддера: Создание главной сюжетной цепочки

В игре есть глобальная сюжетная линия. Игра сама выбирает 4-6 городов, выстраивает между ними маршрут, а игрок должен выполнять задания в каждом из них, чтобы продвигаться дальше. 

Твоя задача — написать **тексты** и задать **тип задания** для каждого этапа этого пути.

## 🟢 ШАГ 1. Открываем "Сценарий"
Весь сюжет живет в файле **`story_script.js`**. Открой его в текстовом редакторе.

Ты увидишь массив `STORY_SCRIPT = [ ... ];`. Каждый объект внутри скобок — это **Этап (Beat)** сюжета, который привязывается к городу по его порядковому номеру (`index`).

*   `index: 0` — Первый город, с которого игрок начинает путь.
*   `index: 1` — Второй город.
*   `index: 2` — Третий город... и так далее.

---

## 🟢 ШАГ 2. Анатомия одного этапа
Каждый этап состоит из 4 параметров. Вот как он выглядит:

```javascript
    {
        index: 0,                     // НОМЕР ЭТАПА (начинается с 0)
        forcedType: 'FETCH',          // ТИП КВЕСТА (что именно нужно сделать)
        customBriefing: "Текст, который NPC говорит, когда дает квест. {item} нужно найти в {location}.",
        customTurnIn: "Текст, который NPC говорит, когда ты сдаешь квест. Спасибо за {item}!"
    },
```

### 🎯 Выбор типа квеста (`forcedType`)
Ты можешь жестко задать, что именно игрок будет делать на этом этапе. Доступные типы:

*   **`'FETCH'`** — Найти конкретный предмет (часто уникальный артефакт) в указанном подземелье.
*   **`'HUNT'`** — Убить определенное количество конкретных монстров (например, 5 Гоблинов) в указанном подземелье.
*   **`'DIGGER'`** — Спуститься на заданную глубину в указанном подземелье (разведка).
*   **`'COLLECT'`** — Собрать несколько предметов (например, 3 Книги или 4 Зелья) в подземелье.
*   **`'EXPLORE'`** — Просто добраться до указанного подземелья и выжить.
*   **`'BOUNTY'`** — Убить указанное количество монстров (где угодно в мире, не привязано к одному подземелью).
*   **`'SCHOLAR'`** — Прочитать определенное количество книг, разбросанных по миру.

💡 *Если ты не укажешь `forcedType`, игра сама случайно выберет тип квеста для этого города!*

---

## 🟢 ШАГ 3. Магия Плейсхолдеров (Заменителей)
Тебе не нужно придумывать названия городов, имена врагов или цены. Игра сама подставит их в текст, если ты используешь **плейсхолдеры** (слова в фигурных скобках).

| Плейсхолдер | Что подставит игра | Пример того, как это увидит игрок |
| :--- | :--- | :--- |
| `{city}` | Название текущего города | *"Старейшина **Эльграда** тяжело опирается..."* |
| `{nextCity}` | Название следующего города | *"Путь в **Темный Холм** будет открыт..."* |
| `{item}` | Название предмета (для FETCH/COLLECT) | *"Найди **Проклятый Амулет**..."* |
| `{enemy}` | Название врага (для HUNT/BOUNTY) | *"Убей **Скелетов**..."* |
| `{count}` | Количество (врагов или предметов) | *"Убей **5** скелетов..."* |
| `{location}` | Название подземелья | *"Они прячутся в **Пещерах Ужаса**..."* |
| `{depth}` | Требуемая глубина (для DIGGER) | *"Спустись на глубину **3**..."* |
| `{gold}` | Награда в золоте | *"Я заплачу тебе **150** золотых."* |

⚠️ **ВАЖНО:** Так как предметы (`{item}`) и враги (`{enemy}`) генерируются случайно (или выбираются из базы), пиши текст так, чтобы он звучал хорошо с **любым** словом. 
*   ❌ *Плохо:* "Найди мой потерянный **меч**..." (А игра подставит "Кольцо" или "Книгу").
*   ✅ *Хорошо:* "Найди мою потерянную **реликвию**..." или "Найди **{item}**..."

## ⚠️ ЗОЛОТЫЕ ПРАВИЛА (Чтобы код не сломался)

Файл `story_script.js` — это JavaScript код. Одна пропущенная запятая может обрушить всю игру.

1. **Запятые между этапами:** После закрывающей фигурной скобки `}` каждого этапа **обязательно** должна стоять запятая `,`.
   * ❌ Ошибка: `} {`
   * ✅ Правильно: `}, {`
2. **Никаких запятых в самом конце:** Перед закрывающей скобкой массива `];` запятая **не нужна**.
3. **Кавычки:** Используй только обычные кавычки `" "`. Внутри текста можно использовать одинарные `' '` для диалогов, чтобы не конфликтовать с двойными.
   * ✅ Правильно: `customBriefing: "Старик сказал: 'Иди домой'."`
4. **Порядок индексов:** Старайся идти по порядку: 0, 1, 2, 3... Если ты пропустишь индекс (например, напишешь 0, 1, 3), то для города под индексом 2 игра использует стандартный процедурный текст.
5. **Длина цепочки:** Игра сама решает, сколько будет городов (от 4 до 6). Если ты напишешь сценарии только для индексов 0, 1 и 2, то для оставшихся городов (3, 4, 5) игра сама сгенерирует случайные квесты из стандартных шаблонов. Это **нормально** и не сломает игру!

Чтобы увеличить количество квестов в сюжетной цепочке, тебе нужно изменить один параметр в файле **`quest_chain.js`**.

Сейчас игра генерирует от **4 до 6** городов для прохождения. Давай сделаем так, чтобы цепочка могла быть длиннее (например, от 5 до 8 этапов).

### 🛠️ Инструкция:

1.  Открой файл **`quest_chain.js`**.
2.  Найди функцию `init(startX, startY)`.
3.  Внутри этой функции найди строку, где создается переменная `chainLength`. Она выглядит так:
    ```javascript
    const chainLength = rng.int(4, 6);
    ```
4.  Измени цифры в скобках на желаемые. Например, чтобы сделать цепочку от 5 до 8 квестов:
    ```javascript
    // Было:
    // const chainLength = rng.int(4, 6);

    // Стало (длинная цепочка):
    const chainLength = rng.int(5, 8);
    ```

### 💡 Важные нюансы:

*   **Детерминизм:** Так как используется `SeededRandom`, изменение этих цифр изменит всю карту мира для этого сида. Старые города останутся на местах, но *порядок* и *количество* следующих городов в цепочке пересчитаются.
*   **Расстояние между городами:** В коде ниже есть поиск следующего города:
    ```javascript
    const candidates = findCitiesInRing(currentCity.x, currentCity.y, 15, 50);
    ```
    Если ты сделаешь цепочку очень длинной (например, 10+), убедись, что на карте достаточно городов. Радиус поиска `15-50` клеток обычно достаточен, но если мир маленький, цепочка может оборваться раньше (`if (candidates.length === 0) break;`).
*   **Тексты квестов:** В файле **`story_script.js`** сейчас прописаны тексты только для первых 5 этапов (`index: 0` ... `index: 4`).
    *   Если цепочка станет длиннее (например, 7 этапов), то для 6-го и 7-го этапа игра автоматически использует **стандартные шаблонные тексты** из `quest_chain.js` (те, что в `CHAIN_TEMPLATES`).
    *   Если ты хочешь уникальные тексты и для новых этапов, просто добавь новые объекты в массив `STORY_SCRIPT` в файле `story_script.js`:
        ```javascript
        {
            index: 5,
            forcedType: 'BOUNTY',
            customBriefing: "Твой путь лежит через земли варваров...",
            customTurnIn: "Варвары уважают силу. Иди дальше."
        },
        ```

После изменения сохрани файл и обнови страницу в браузере (F5), чтобы сгенерировать новую цепочку с новыми параметрами.

# 🕸️ V. Руководство моддера: Интеграция Twine-приключений

Текстовые квесты (Twine) — это способ добавить в игру глубокие сюжетные диалоги, моральные выборы и уникальные награды, которые невозможно реализовать через обычный генератор подземелий. 

Тебе **не нужно** лезть в `index.html` или `game.js`. Вся магия происходит прямо в тексте твоего приключения с помощью специальных вставок кода. Сами приключения в формате Twine можно писать, не зная ни строчки кода: достаточно освоить простой синтаксис скриптов в формате Twee, а создавать текстовые приключения можно в онлайн- и оффлайн редакторе Twine. В них можно играть и без основной игры, потому что они экспортируются в виде отдельных html-файлов. Но наша идея заключается в том, чтобы включить созданные текстовые приключения в мир нашей roguelike, для чего в сценарии Twee будут добавляться специальные ссылки и фрагменты javascript, отвечающие за передачу информации в основной код игры. 

---

## 🟢 ЧАСТЬ 1. Как Twine общается с игрой?

Когда игрок нажимает на специальную ссылку в Twine-квесте, срабатывает скрытый JavaScript-код. Он отправляет "послание" (payload) основной игре: *"Эй, квест завершен! Вот награда!"*. 
Игра получает послание, начисляет награду и **автоматически закрывает окно Twine**, возвращая игрока в подземелье.

### Базовый синтаксис "послания"
Вот тот самый сниппет, который ты будешь вставлять в свои квесты:

```html
<script>
    window.parent.postMessage({
        type: 'TWINE_QUEST_COMPLETE',
        payload: { 
            // СЮДА ПИШЕМ НАГРАДЫ 
        } 
    }, '*');
</script>
```

### Куда вставлять этот код?
В редакторе Twine (или в формате Twee) у тебя есть два пути:

**Путь А. Код срабатывает по клику на ссылку (Рекомендуется)**
Игрок читает текст, нажимает на ссылку-кнопку, и только тогда получает награду. Код оборачивается в `(link: "...")[ ... ]`.
```twee
Старик радостно улыбается и протягивает тебе кошелек.

(link: "Забрать золото и уйти")[
    <script>
        window.parent.postMessage({
            type: 'TWINE_QUEST_COMPLETE',
            payload: { gold: 50, message: "Старик благословляет вас." } 
        }, '*');
    </script>
]
```

**Путь Б. Код срабатывает мгновенно (Автоматический конец)**
Если ты хочешь, чтобы квест завершился сразу, как только игрок дочитал узел до конца (без нажатия на кнопку).
```twee
Дверь захлопнулась перед вашим носом. Придется искать другой путь.

<script>
    window.parent.postMessage({
        type: 'TWINE_QUEST_COMPLETE',
        payload: { gold: 0 } 
    }, '*');
</script>
```

---

## 🟢 ЧАСТЬ 2. Справочник по наградам (Payload)

Внутри `payload: { ... }` ты можешь комбинировать любые награды. Вот полный список того, что понимает игра:

### 💰 1. Золото (`gold`)
Изменяет количество золота у игрока.
*   `gold: 100` — Добавить 100 монет.
*   `gold: -50` — Отнять 50 монет (например, если игрок был обокраден или заплатил штраф).
*   `gold: 0` — Ничего не менять (квест завершен без денежной награды).

### ✨ 2. Опыт (`xp`)
Дает игроку опыт для повышения уровня.
*   `xp: 50` — Дадет 50 единиц опыта.

### ❤️ 3. Лечение (`healPercent`)
Восстанавливает здоровье в процентах от максимального HP.
*   `healPercent: 0.5` — Восстановит 50% от макс. HP.
*   `healPercent: 1.0` — Полностью исцелит игрока.

### ⚡ 4. Выносливость (`stamina`)
Восстанавливает или тратит выносливость (расходуется на глобальной карте).
*   `stamina: 30` — Добавит 30 единиц выносливости.
*   `stamina: -1` — **Полностью восстановит** выносливость до максимума (эффект хорошего отдыха).

### 🎁 5. Выдача уникального предмета (`itemId`)
Добавляет в инвентарь уникальный артефакт из базы `UNIQUE_ITEM_TEMPLATES` (файл `data.js`).
*   `itemId: "unique_weapon_excalibur"` — Выдаст "Святой Меч".
*   `itemId: "unique_item_ring_power"` — Выдаст "Кольцо Всевластия".
*(ID предмета нужно смотреть в `data.js` в массиве `UNIQUE_ITEM_TEMPLATES`)*.

### 🗑️ 6. Удаление предмета (`removeItem`)
Забирает предмет из инвентаря игрока (поиск идет по названию).
*   `removeItem: "Зелье лечения"` — Заберет первое найденное зелье лечения.
*   `removeItem: "Кольцо"` — Заберет любой предмет, в имени которого есть слово "Кольцо".

### 🚩 7. Сюжетный флаг (`questFlag`)
Устанавливает невидимый "флаг" в памяти игры. Это полезно, если ты делаешь серию квестов, и второй квест должен знать, что игрок сделал в первом.
*   `questFlag: "saved_the_old_man"` — Игра запомнит, что ты спас старика.
*(В коде игры это можно будет проверить через `GameModule.getGlobalFlag("saved_the_old_man")`)*.

### 💬 8. Сообщение в лог (`message`)
Выводит красивый текст в игровое окно событий (справа внизу).
*   `message: "Вы почувствовали странное покалывание в пальцах."`

## 🟢 ЧАСТЬ 3. Готовые шаблоны (Копируй и меняй!)

### 📜 Шаблон 1: Классическая награда (Золото + Опыт + Текст)

## 🟢 ЧАСТЬ 4. Синтаксис Twee (Краткая шпаргалка)

Если ты пишешь квест в формате Twee (текстовый файл) или просто хочешь понять логику Twine (Harlowe):

1. **Узлы (Passages)** начинаются с двух двоеточий:
   ```twee
   :: Начало
   Текст начала...
   
   :: Концовка_А
   Текст концовки...
   ```
2. **Ссылки-переходы** создаются так:
   ```twee
   [[- Пойти налево->Налево]]
   [[- Пойти направо->Направо]]
   ```
3. **Переменные** (если нужно считать что-то внутри Twine):
   ```twee
   (set: $gold_stolen to 0)
   (set: $gold_stolen to $gold_stolen + 50)
   (if: $gold_stolen > 100)[Вы богаты!]
   ```
   *Но помни: переменные Twine живут только внутри окна Twine. Чтобы передать данные в основную RPG-игру, используй `payload`!*

---

## ⚠️ Чек-лист перед сохранением

1. **Кавычки в payload:** Используй только одинарные `' '` или двойные `" "` кавычки. Не используй «ёлочки».
   * ❌ `payload: { message: «Привет» }`
   * ✅ `payload: { message: "Привет" }`
2. **Запятые:** Разделяй награды запятыми.
   * ❌ `payload: { gold: 50 xp: 10 }`
   * ✅ `payload: { gold: 50, xp: 10 }`
3. **Одинарные кавычки внутри текста:** Если в сообщении есть цитата, используй внутри одинарные кавычки, чтобы не сломать двойные.
   * ✅ `message: "Старик крикнул: 'Беги!' и упал."`
4. **Закрытие скобок:** Убедись, что у `<script>` есть закрывающий `</script>`, а у `payload` есть закрывающие `}` и `}`.
5. **Автоматическое закрытие:** Помни, что после срабатывания `postMessage` окно Twine **закроется само**. Не пиши в тексте "Нажмите сюда, чтобы закрыть окно".
6. Текстовые квесты Twine, сохраненные из редактора Twine в html-формате, нужно хранить в корне проекта игры, чтобы игра могла быстро к ним обращаться.


Отличная идея! Это критически важный шаг. Можно написать гениальный сюжет, но если игра не будет знать о существовании файла, игрок его никогда не увидит.

Вот готовый блок, который логично вставить в гайд по Twine-квестам (например, как **ЧАСТЬ 6. Регистрация и запуск приключений в игре**).

## 🟢 ЧАСТЬ 5. Как добавить свое Twine-приключение в игру (Регистрация и Запуск)

Ты написал историю, протестировал её в редакторе Twine, и экспортировал в формат **HTML**. Отлично! Но игра пока не знает, что этот файл нужно показывать игроку. Давай научим игру подгружать твоё приключение.

### 📁 Шаг 1. Правильное размещение файла
Твой экспортированный HTML-файл (например, `My_Cool_Quest.html`) должен лежать **в той же папке**, что и `index.html` (корневая папка игры). 
*   ❌ Нельзя прятать его в подпапки вроде `quests/` или `twine/`.
*   ✅ Просто кидай его в корень.

### 📝 Шаг 2. Прописка в "Ростере" (Списке квестов)
Чтобы игра начала использовать твой файл, его нужно добавить в специальный список.

1. Открой файл **`npc_generator.js`** в текстовом редакторе.
2. В самом начале найди массив `TEXT_QUESTS_ROSTER`. Он выглядит так:
   ```javascript
   const TEXT_QUESTS_ROSTER = [
       'Quack of Duckness.html',
       // Сюда можно добавить другие файлы, когда они появятся:
       // 'The_Haunted_Mansion.html',
       'RingOfPower.html'
   ];
   ```
3. Добавь имя своего файла в этот список. **Не забудь про кавычки и запятую!**
   ```javascript
   const TEXT_QUESTS_ROSTER = [
       'Quack of Duckness.html',
       'RingOfPower.html',
       'My_Cool_Quest.html' // <--- ТВОЙ НОВЫЙ ФАЙЛ
   ];
   ```
*⚠️ Важно: Имя файла в коде должно **точно-в-точно** совпадать с именем файла на диске, включая регистр (большие/маленькие буквы) и пробелы.*

---

### ⚙️ Шаг 3. Принцип запуска (Как игрок получит твой квест?)

Ты мог заметить, что в игре нет отдельной кнопки "Взять текстовый квест". Система их выдачи **встроена в генерацию городов**. Вот как это работает "под капотом":

1. **Загадочная фигура:** Когда игрок заходит в новый город, игра с вероятностью **80%** спавнит там особого NPC — *"Загадочную фигуру"* (она светится фиолетовым цветом `#ff00ff`).
2. **Правило одного города:** Один город может выдать **только один** текстовый квест за всю игру. Если игрок уже поговорил с фиолетовым NPC в этом городе (или отказался от него), этот город больше никогда не выдаст текстовый квест.
3. **Память о пройденном:** Игра помнит, какие именно HTML-файлы игрок уже завершил (от начала до конца). 
   * Если игрок завершил `My_Cool_Quest.html`, игра вычеркнет его из списка доступных.
   * Фиолетовый NPC будет предлагать только те приключения, которые игрок **ещё не проходил**.
4. **Исчезновение:** Как только игрок завершает квест (или закрывает окно крестиком), NPC физически удаляется из города, чтобы игрок не мог "фармить" золото, разговаривая с ним повторно.

💡 **Лайфхак для баланса:** Если ты добавил в игру 5 крутых текстовых квестов, игроку потребуется посетить минимум 5 разных городов, чтобы собрать их все. Это отлично мотивирует исследовать глобальную карту!

---

### 🛠️ Шаг 4. Как быстро протестировать свой квест (Режим отладки)

Тебе не обязательно каждый раз бегать по карте и искать города, чтобы проверить своё приключение. В коде игры уже есть "чит-кнопка" для быстрого запуска!

В файле **`game.js`** найди блок обработки клавиш (внутри функции `handleInput`):
```javascript
// Временная клавиша для теста Twine
if (e.key === 'k' || e.key === 'K') {
    e.preventDefault();
    GameModule.openTwineQuest('Quack of Duckness.html');
    return;
}
```

**Как использовать:**
Просто замени `'Quack of Duckness.html'` на имя своего нового файла, сохрани `game.js`, обнови страницу (F5) и нажми клавишу **`K`** (или `k` / `л` / `L` на русской раскладке, если она не перехватывается). 
Твоё приключение мгновенно откроется поверх карты!

*(Не забудь вернуть всё как было или закомментировать строку, когда закончишь тесты, чтобы кнопка `K` не мешала в будущем).*

### ⚠️ Чек-лист перед публикацией приключения

1. [ ] Файл экспортирован из Twine именно в формате **HTML** (для веб-браузера).
2. [ ] Файл лежит в **корневой папке** игры (рядом с `index.html`).
3. [ ] Имя файла прописано в массиве `TEXT_QUESTS_ROSTER` внутри `npc_generator.js`.
4. [ ] В имени файла **нет специальных символов** (лучше использовать `_` вместо пробелов, например `Haunted_Mansion.html`, хотя игра понимает и пробелы).
5. [ ] Внутри HTML-файла в нужных местах стоят скрипты отправки наград (`window.parent.postMessage...`).
6. [ ] Файл сохранен в кодировке **UTF-8** (иначе русский текст может превратиться в "кракозябры" `РєРєРєС‚`). В Twine это обычно по умолчанию, но если редактировал в Блокноте — проверь.

Далее расскажем про сюжетные триггеры или "флаги", которые можно менять внутри Twine-приключений и передавать из значения в основную игру.
Сюжетные флаги (Flags) — это самый мощный инструмент для создания **нелинейности** и **последствий выбора** в твоей игре. 

Флаг — это невидимая "галочка" в памяти игры. Если игрок проявил милосердие в Twine-квесте, игра ставит галочку. А потом, через 10 часов геймплея, другой NPC или книга могут "вспомнить" этот поступок.

## 🚩 ЧАСТЬ 7. Сюжетные Флаги (Как игра помнит твой выбор)

Иногда мало просто дать игроку золото или предмет. Хочется, чтобы игра **запомнила** моральный выбор или важное событие, и это повлияло на мир в будущем. Для этого используются **Флаги (Flags)**.

### Как это работает?
1. В Twine-квесте ты отправляешь игре сигнал: *"Запомни, что игрок спас старика"*.
2. Игра ставит невидимую галочку (флаг) в своей памяти.
3. В любом другом месте игры (в диалогах NPC, в книгах, в генерации квестов) ты можешь написать условие: *"Если галочка стоит — веди себя иначе"*.

### Шаг 1. Ставим Флаг в Twine
Внутри `payload` добавь параметр `questFlag` и придумай для него уникальное имя на английском (без пробелов).

```twee
(link: "Отдать старику последний золотой")[
    <script>
        window.parent.postMessage({
            type: 'TWINE_QUEST_COMPLETE',
            payload: { 
                gold: -10, 
                questFlag: "mercy_to_old_man", // <--- ИГРА ЗАПОМНИТ ЭТО
                message: "Старик благословляет тебя. Вы чувствуете странное тепло в груди." 
            } 
        }, '*');
    </script>
]
```
💡 **Совет по именам:** Используй стиль `snake_case` (слова через нижнее подчеркивание). Например: `killed_grigoriy`, `stole_amulet`, `saved_the_village`. Имена чувствительны к регистру: `Saved_Village` и `saved_village` — это **два разных флага**!

`GameModule.getGlobalFlag("имя_флага")` (вернет `true`, если флаг установлен).

### ⚠️ Важные правила работы с Флагами

1. **Флаги вечны (почти).** Если ты поставил флаг `met_king`, он будет храниться в памяти игры до тех пор, пока игрок не нажмет F5 (полный перезапуск) или не очистит кэш браузера. Это идеально для мета-сюжета!
2. **Очистка флага.** Если событие должно произойти *только один раз* (как в примере с Наследником старика), не забудь "стереть" флаг после срабатывания:
   ```javascript
   window.globalFlags['my_flag'] = false; // или delete window.globalFlags['my_flag'];
   ```
3. **Проверка на существование.** Всегда оборачивай проверку флага в `if (typeof GameModule !== 'undefined' ...)`. Это спасет от ошибок, если модули игры еще не загрузились.
4. **Комбинирование.** Ты можешь проверять несколько флагов сразу, создавая сложные ветки сюжета:
   ```javascript
   if (GameModule.getGlobalFlag('killed_dragon') && !GameModule.getGlobalFlag('saved_princess')) {
       // Мир узнал, что дракон мертв, но принцесса погибла вместе с ним!
       // Открываем мрачную версию концовки...
   }
   ```

Подземелья — это сердце игры. Каждый раз, когда игрок спускается под землю, алгоритм рисует уникальную карту: комнаты, коридоры, пещеры, ледяные гроты. В этом гайде ты научишься **менять внешний вид** существующих подземелий, **настраивать их параметры** и даже **создавать свои собственные типы**!

## 🟢 ШАГ 2. Анатомия типа подземелья

```javascript
{ 
    name: 'cave',           // ВНУТРЕННЕЕ ИМЯ (используется в коде, не меняется игроком)
    weight: 15,             // ВЕС (шанс выпадения). Чем больше, тем чаще встречается
    emoji: '🕸️',            // ЭМОДЗИ (для отладки и логов)
    floorChar: 'o',         // СИМВОЛ ПОЛА (используется для выбора спрайта из тайлсета)
    wallChar: 'O',          // СИМВОЛ СТЕНЫ
    floorColor: '#2a2a2a',  // ЦВЕТ ПОЛА (если спрайт не найден — будет этот цвет)
    wallColor: '#4a3b3b'    // ЦВЕТ СТЕНЫ
}
```

### 🎯 Что такое `weight` (вес)?
Вес определяет, **как часто** этот тип будет появляться на глобальной карте. Игра суммирует все веса и выбирает тип по принципу рулетки.

**Текущая статистика (сумма весов = 100):**
| Тип | Вес | Шанс |
|---|---|---|
| `dungeon` (классика) | 30 | 30% |
| `icy` (лёд) | 20 | 20% |
| `rogue` (руины) | 20 | 20% |
| `cave` (пещера) | 15 | 15% |
| `cellular` (органика) | 10 | 10% |
| `arena` (арена) | 3 | 3% |
| `boss` (логово босса) | 2 | 2% |

💡 **Хочешь больше пещер?** Увеличь `weight` у `cave` до 30.  
💡 **Хочешь реже видеть боссов?** Уменьши `weight` у `boss` до 1.

## 🟢 ШАГ 5. Алгоритмы генерации (как устроены подземелья)

В файле **`dungeon_generator.js`** есть 4 основных алгоритма. Понимая их, ты сможешь тонко настраивать любой тип.

### 🏠 Алгоритм 1: Комнаты и коридоры (`generateRoomCorridorMap`)
**Используется для:** `dungeon`, `icy`, `rogue`, `boss`, `arena`

**Как работает:**
1. Создаёт пустую карту, заполненную стенами.
2. Раскидывает по ней случайные **прямоугольные комнаты**.
3. Соединяет комнаты **коридорами**.

**Ключевые параметры (внутри функции):**
```javascript
const roomCount = rand.int(10, 20);  // Количество комнат (от 10 до 20)
const w = rand.int(4, 8);            // Ширина комнаты (от 4 до 8 клеток)
const h = rand.int(4, 8);            // Высота комнаты (от 4 до 8 клеток)
```

💡 **Хочешь больше комнат?** Измени `rand.int(10, 20)` на `rand.int(25, 40)`.  
💡 **Хочешь огромные залы?** Измени `rand.int(4, 8)` на `rand.int(8, 15)`.

### 🕳️ Алгоритм 2: Пещеры (`generateCaveMap`)
**Используется для:** `cave`

**Как работает:**
1. Заполняет карту случайным шумом (каждая клетка — стена с шансом 45%).
2. Применяет **5 итераций сглаживания** (клеточный автомат) — шум превращается в плавные пещеры.
3. Находит самые большие регионы и соединяет их тоннелями.

**Ключевые параметры:**
```javascript
const fillChance = 0.45;   // Шанс, что клетка станет стеной (0.45 = 45%)
for (let i = 0; i < 5; i++) {  // Количество итераций сглаживания
    grid = smoothMap(grid, width, height);
}
```

💡 **Хочешь более "дырявые" пещеры?** Уменьши `fillChance` до `0.35`.  
💡 **Хочешь более гладкие, округлые пещеры?** Увеличь количество итераций сглаживания до `7`.

### 🧬 Алгоритм 3: Клеточный автомат (`generateCellularMap`)
**Используется для:** `cellular`

**Как работает:** Похож на пещеры, но с другим правилом сглаживания. Создаёт "органические", похожие на живые ткани структуры.

**Ключевые параметры:**
```javascript
const fillChance = 0.45;
for (let iter = 0; iter < 4; iter++) {  // 4 итерации
    // ...
    if (wallCount >= 5) newGrid[y][x] = 1;  // Если вокруг 5+ стен — становимся стеной
}
```

💡 **Хочешь более "плотные" структуры?** Измени порог `wallCount >= 5` на `wallCount >= 4`.

### 🏟️ Алгоритм 4: Арена (`generateArenaMap`)
**Используется для:** `arena`, `boss`

**Как работает:**
1. Создаёт **одну огромную открытую комнату** (почти вся карта — пол).
2. Раскидывает по ней случайные **колонны** (стены 1x1 или 2x2).

**Ключевые параметры:**
```javascript
const margin = 2;  // Отступ от краёв карты (стены по периметру)
const colCount = rand.int(5, 15);  // Количество колонн
```

💡 **Хочешь больше укрытий на арене?** Измени `rand.int(5, 15)` на `rand.int(20, 40)`.

## 🟢 ШАГ 6. Создаём свой уникальный алгоритм

Допустим, ты хочешь, чтобы **"Проклятый храм"** генерировался как **крестовидная комната** с колоннами по углам. Давай напишем свою функцию!

### Шаг 6.1. Пишем функцию генерации

В файле **`dungeon_generator.js`** добавь новую функцию рядом с другими (например, после `generateArenaMap`):

```javascript
// === ГЕНЕРАЦИЯ КРЕСТОВИДНОГО ХРАМА ===
function generateTempleMap(rand, width, height) {
    // 1. Начинаем с карты, полностью заполненной стенами
    const grid = Array(height).fill().map(() => Array(width).fill(1));
    
    // 2. Рисуем "крест" в центре карты
    const centerX = Math.floor(width / 2);
    const centerY = Math.floor(height / 2);
    
    // Горизонтальная перекладина креста
    const armWidth = rand.int(15, 25);   // Длина горизонтальной части
    const armHeight = rand.int(6, 10);   // Ширина горизонтальной части
    for (let y = centerY - armHeight/2; y < centerY + armHeight/2; y++) {
        for (let x = centerX - armWidth/2; x < centerX + armWidth/2; x++) {
            if (y >= 0 && y < height && x >= 0 && x < width) {
                grid[y][x] = 0;  // 0 = пол
            }
        }
    }
    
    // Вертикальная перекладина креста
    const vertWidth = rand.int(6, 10);
    const vertHeight = rand.int(15, 25);
    for (let y = centerY - vertHeight/2; y < centerY + vertHeight/2; y++) {
        for (let x = centerX - vertWidth/2; x < centerX + vertWidth/2; x++) {
            if (y >= 0 && y < height && x >= 0 && x < width) {
                grid[y][x] = 0;
            }
        }
    }
    
    // 3. Добавляем колонны по углам креста
    const pillarPositions = [
        { x: centerX - armWidth/2 + 2, y: centerY - vertHeight/2 + 2 },
        { x: centerX + armWidth/2 - 2, y: centerY - vertHeight/2 + 2 },
        { x: centerX - armWidth/2 + 2, y: centerY + vertHeight/2 - 2 },
        { x: centerX + armWidth/2 - 2, y: centerY + vertHeight/2 - 2 }
    ];
    
    for (const pos of pillarPositions) {
        // Колонна 2x2
        for (let dy = 0; dy < 2; dy++) {
            for (let dx = 0; dx < 2; dx++) {
                const px = pos.x + dx;
                const py = pos.y + dy;
                if (px >= 0 && px < width && py >= 0 && py < height) {
                    grid[py][px] = 1;  // 1 = стена (колонна)
                }
            }
        }
    }
    
    return grid;
}
```

### Шаг 6.2. Подключаем функцию к типу подземелья

Найди в `dungeon_generator.js` функцию `generateLevelWithType`. Внутри неё есть блок `if-else`, который выбирает алгоритм по имени типа:

```javascript

    const caveResult = generateCaveMap(rand, width, height);
    mapGrid = caveResult.grid;
    startPos = caveResult.startPos;
} else if (dungeonType.name === 'cellular') {
    mapGrid = generateCellularMap(rand, width, height);
} else if (dungeonType.name === 'arena' || dungeonType.name === 'boss') {
    mapGrid = generateArenaMap(rand, width, height);
} else {
    mapGrid = generateRoomCorridorMap(rand, width, height);  // Стандарт
}
```

Добавь свою проверку **перед** последним `else`:

```javascript

    mapGrid = generateTempleMap(rand, width, height);  // Твоя функция!
} else {
    mapGrid = generateRoomCorridorMap(rand, width, height);
}
```

## 🟢 ШАГ 7. Привязка типа к глобальной карте

По умолчанию все типы подземелий могут появиться в **любом месте** глобальной карты. Но что, если ты хочешь, чтобы **"Проклятый храм"** появлялся **только далеко от старта**?

Открой файл **`globalMap.js`**, найди функцию `generatePOIs`. Внутри есть блок генерации входов в подземелья:

```javascript
// 2. Входы в подземелья
for (let y = 0; y < height; y++) {
    for (let x = 0; x < width; x++) {
        const currentTile = tiles[y][x];
        const isValidTerrain = (currentTile === 'plain' || currentTile === 'forest' || currentTile === 'road');
        
        if (isValidTerrain && rand.next() < GLOBAL_CONFIG.DUNGEON_DENSITY) {
            if (currentTile !== 'city') {
                if (isTooClose(x, y)) continue;
                
                tiles[y][x] = 'dungeon_entrance';
                const globalX = cx * width + x;
                const globalY = cy * height + y;
                
                const dungeonType = DungeonGeneratorModule.getRandomDungeonType(rand).name;
                const { fullName } = NameGeneratorModule.generateLocationData(globalX, globalY, dungeonType);
                pois.push({ x: globalX, y: globalY, type: 'dungeon', dungeonType: dungeonType, name: fullName });
            }
        }
    }
}
```

Чтобы твой `temple` появлялся только на глубине 50+ клеток от центра мира, добавь проверку **перед** выбором типа:

```javascript
const dungeonType = DungeonGeneratorModule.getRandomDungeonType(rand).name;

// === НОВОЕ: Фильтр по расстоянию для храма ===
const distFromCenter = Math.abs(globalX) + Math.abs(globalY);
if (dungeonType === 'temple' && distFromCenter < 50) {
    continue; // Пропускаем эту клетку, храм здесь не появится
}
// ==========================================

const { fullName } = NameGeneratorModule.generateLocationData(globalX, globalY, dungeonType);
```

💡 **Идеи для привязки:**
*   `arena` и `boss` — только далеко от старта (сложные типы)
*   `cellular` — только в "лесных" чанках (проверка `tiles[y][x] === 'forest'`)
*   `icy` — только в "горных" регионах (рядом с `mountain`)

## ⚠️ Чек-лист перед сохранением

1. **Уникальное имя:** Проверь, что `name` у твоего нового типа не совпадает с существующими (`dungeon`, `cave`, `icy`, `rogue`, `cellular`, `arena`, `boss`).
2. **Существующие символы:** Убедись, что `floorChar` и `wallChar` есть в `sprite_registry.js` и `tileset_renderer.js`. Иначе увидишь жёлтые/красные квадраты ошибок.
3. **Запятые:** В массиве `DUNGEON_TYPES` после каждой `}` должна быть `,`. Перед `];` — запятая не нужна.
4. **Добавил в name_generator.js?** Если создал новый тип — не забудь добавить для него названия в `locationTypes`.
5. **Подключил алгоритм?** Если написал свою функцию генерации — добавь проверку `else if (dungeonType.name === 'твой_тип')` в **обе** функции: `generateLevel` и `generateLevelWithType`.
6. **Кавычки:** Используй только `" "` и `' '`, а не «ёлочки».
7. **Тест веса:** Если хочешь быстро проверить свой тип — временно поставь `weight: 1000`, а остальным типам — `weight: 0`. После теста верни нормальные значения.
